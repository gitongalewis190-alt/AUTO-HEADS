Section 2 — Register: Step 1 (Live)

Step indicator dots · Live @username preview · Password strength meter · Confirm password green tick. Try typing a weak password like "abc" then a strong one like "MyC@r2026!".

In plain English

Sign-up happens in 3 short steps instead of one long form, which feels less intimidating. Three dots near the top show progress — the current step stretches into a long orange pill, finished steps turn green. As someone types a username, a live preview shows exactly what their public profile link will look like (e.g. autoheads.co/@david_abel). As they type a password, a coloured bar fills up — red means weak, green means strong — so they know immediately if it's safe, before they even submit.

Section 3 — Register: Step 2 — Phone OTP (Live)

OTP boxes auto-advance on each digit. Backspace goes to the previous box. Paste a 6-digit number to fill all at once. Try pasting 123456.

In plain English

Instead of one long box for the 6-digit code sent by SMS, there are 6 separate boxes. Type a digit and the cursor automatically jumps to the next box — no tapping needed between digits. Hit backspace on an empty box and it jumps back to fix the previous one. If the code arrives as a text message, the whole 6-digit number can be copied and pasted in one go — it fills every box instantly instead of one at a time. A countdown tells the person how long until they're allowed to ask for a new code.

Section 5 — Button System

Three levels. .btn-primary for the main action. .btn-secondary for alternatives (Google, etc.). .btn-ghost for low-priority (back, forgot). .btn-guest for the dashed guest button.

In plain English

Every button's style tells you how important it is, at a glance. Solid orange means "this is the main thing to do here" (Sign In, Create Account). A glass-bordered button means "here's another way to do it" (Continue with Google). Plain text means "low priority, do this only if you need to" (Back, Forgot password?). A dashed outline means "this is an alternative path" — browsing as a guest instead of signing up.

Section 6 — Feedback States

Success badge (.success-badge), error block (.global-error), and field-level error (.field-error). All use animation: fadeIn.

In plain English

Good news always appears in a green box, bad news always in a red box — the colour alone tells you which it is before you even read the words. Both fade in gently rather than popping in suddenly, which feels calmer and less jarring than an instant flash.

Appendix — Quick Reference (Plain English)

A one-page summary of every technique above, for anyone who just wants the gist without scrolling back up.

How everything works

1. Background orbs

Soft glowing colour blobs in the corners, blurred so they read as ambient light, not shapes.

2. Frosted-glass panel

The form card looks like foggy glass with a slowly rotating glow around its edge.

3. Glowing inputs

Tapped fields light up orange; wrong entries shake and turn red; correct ones turn green.

4. Password strength bar

A bar fills and changes colour live as you type — red (weak) to green (strong).

5. 6-box OTP code

One digit per box, auto-jumps forward, backspace jumps back, paste fills all six at once.

6. Buttons by importance

Solid orange = main action. Glass border = alternative. Plain text = low priority. Dashed = guest path.

7. Step dots

Three dots track sign-up progress; the current step stretches and glows, finished steps turn green.

8. Success / error messages

Green always means it worked, red always means something needs fixing — colour-coded before you even read the text.
